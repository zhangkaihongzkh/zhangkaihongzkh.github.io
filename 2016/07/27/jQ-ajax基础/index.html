JQ-ajax基础

发表于 2016-07-27   |   分类于 JQ基础篇   |   | 阅读量 次

摘要:1.ajax的优缺点 2.load()方法 3.$.get()方法和$.post() 4.$.getScript()和$.getJSON() 5.$.ajax()方法 6.表单序列化

jQuery对Ajax进行了大量的封装，我们不需要考虑浏览器的兼容性。
其中JQ进行了三层封装
最底层是$.ajax()
通过上层封装了第二层.load(),$.get()和$.post
最高层为$.getScript和$.getJSON()

ajax的优缺点

优点

1.不需要插件支持（一般浏览器且默认开启 JavaScript 即可）
2.用户体验极佳（不刷新页面即可获取可更新的数据）
3.提升 Web 程序的性能（在传递数据方面做到按需放松，不必整体提交）
4.减轻服务器和带宽的负担（将服务器的一些操作转移到客户端）

缺点

1.不同版本的浏览器度 XMLHttpRequest 对象支持度不足(比如 IE5 之前)；
2.前进、后退的功能被破坏（因为 Ajax 永远在当前页，不会几率前后页面） ；
3.搜索引擎的支持度不够（因为搜索引擎爬虫还不能理解 JS 引起变化数据的内容） ；
4.开发调试工具缺乏（相对于其他语言的工具集来说，JS 或 Ajax 调试开发少的可怜） 。

$.get方法与$.post方法区别

1.GET 请求是通过 URL 提交的，而 POST 请求则是 HTTP 消息实体提交的；
2.GET 提交有大小限制（2KB） ，而 POST 方式不受限制；
3.GET 方式会被缓存下来，可能有安全性问题，而 POST 没有这个问题；
4.GET 方式通过$_GET[]获取，POST 方式通过$_POST[]获取

表单序列化

Ajax用的比较多的地方是表单操作，传统的表单操作时通过submit提交将数据传输到服务器端。如果使用ajax异步处理的话，我们需要将每个表单元素逐个获取方能提交。这样工作效率大大降低
