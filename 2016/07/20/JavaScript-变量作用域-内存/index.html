JavaScript 变量 作用域 内存

摘要:1.变量和作用域 2.内存问题

变量和作用域

基本类型和引用类型

基本类型

  • Undefined
  • Null
  • Boolean
  • Number
  • String

这些变量的值保存在栈空间中，我们通过按值来查找

引用类型

如果赋值为引用类型,则需保存在堆内存中,而赋值大小不固定,所以保存引用地址在栈内存中,当查找变量时,先访问栈内存的引用地址,再访问堆内存中的值,这种方式称为按引用访问

动态属性

定义基本属性类型值和引用类型是相似的: 创建一个变量并为该变量赋值，但不同类型值操作会有不同

  • 引用类型

  • 基本类型

复制变量值

在复制变量值方面 基本类型是复制值本身,而引用类型是复制地址

  • 基本类型


box2虽然是box的一个副本，但是两者是独立开的，对两者的变量的操作不影响

  • 引用类型


在引用类型中box2其实就是box,他们指向同一个对象Object 如果Object的name属性被修改了,对应的输出值也会被修改

传递参数

ECMAScript中所有函数都是按值传递的,虽然变量有引用类型和基本类型之分

  • 按值传递

  • 按引用传递

如果存在按引用传递的话，那么函数里的那个变量将会是全局变量，在外部也可以访问

类型检测

当我们要检测变量类型时,可以使用typeof运算符来判断

typeof检测基本数据类型时比较有用,但是检测引用类型时不方便,因为我们不想知道他 是不是对象 ,而是想知道他是 什么类型的对象 因为数组是Object，null也是Object。 这时候我们就要用 instanceof 运算符了

当用instanceof检测基本类型时，会返回 false。

执行环境和作用域

JavaScript的执行环境定义变量或者函数有权访问的其他数据，并决定一些特定行为
全局执行环境是最外围的执行环境，在Web浏览器中规定全局执行环境为 window对象 。所有 全局变量和函数 都是作为window的 属性和和方法 创建的。

全局变量和函数

PS：当执行环境中所有代码执行完后就被销毁，如果为全局执行环境，则要等待程序执行结束或者网页关闭才会销毁
PS：每一个执行环境都有对应的变量对象，就像全局执行环境有window对象，局部也有，但是我们无法访问（解析器处理数据时后台使用它）

函数里的局部作用域变量替换全局变量，但是作用域仅在该局部环境中

通过传参可以替换函数体内的局部变量，但作用域仅在该函数体中

函数体内包含函数，只有这个函数可以访问内一层的函数

PS：每个函数在执行时都会创建自己的执行环境，当执行到这个函数时，函数的执行环境就会推到环境栈中，当函数执行完毕后，会退出环境栈，同时将控制权交给上一级执行环境
PS：当代码在一个执行环境中时，就会形成一种叫 作用域链 的东西。他的用途是 保证对执行环境中有权限访问的变量和函数进行有序的访问 。作用域链的前端，就是执行环境的变量对象

没有块级作用域

级作用域表示诸如 if语句等有花括号封闭的代码块，所以，支持条件判断来定义变
量。

for也是类似

var 关键字在函数中的区别

一般确定变量都是通过搜索来确定该标识符实际代表什么


PS：查询变量中，访问局部变量比访问全局变量要快，所以不需要向上搜寻作用域链。

内存问题

一般来说，确保占用最少的内存可以让页面获得更好的性能。优化内存的最佳方案，
就是一旦数据不再有用，那么将其设置为null来释放引用，这个做法叫做 解除引用。这一
做法适用于大多数全局变量和全局对象
