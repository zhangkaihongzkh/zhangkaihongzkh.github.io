新闻管理平台

技术选型

前端：

– Bootstrap

后端：

– PHP

数据库：

– MYSQL

文件结构

  • dist
    • 存放压缩混淆过后的文件
  • fonts
    • 平台用到的字体
      -images
    • 图片
  • lib
    • 库与框架
  • node_modules
    • node模块
  • src
    • JS与LESS

打包工具

  • gulpfile
    • 打包流程构建工具

逻辑处理

  • success.php
    • 用于处理成功的逻辑判断(传参：$message,$url)
  • error.php
    • 用于处理失败的逻辑判断(传参：$message)
  • login_check.php
    • 用于用户和管理员登录检测
  • register_check.php
    • 用于用户和管理员注册检测

页面模板

用户部分

  • news.php
    • 用户登录首页
  • news_header.php
    • 用户界面头部
  • news_aside
    • 用户界面侧边栏
  • news_footer.php
    • 用户界面底部
  • content.php
    • 具体新闻模板

管理员部分

  • manage.php
    • 管理员首页
  • manage_header.php
    • 管理员界面头部
  • manage_aside
    • 管理员界面侧边栏
  • manage_footer.php
    • 管理员界面底部
  • add.php
    • 管理员增加新闻模板
  • edit.php
    • 管理员修改新闻模板

功能分析

用户

  • 登录注册功能
  • 查看新闻
  • 发表评论

管理员

  • 登录注册功能
  • 发布新闻
  • 删除新闻
  • 修改新闻

模块实现

登录注册


前端：

  • vaidate表单校验用户只能用手机号注册 管理员能用账号注册

后端：

  • login_check.php:用于用户和管理员登录检测
    • 构建查找 从数据库中匹配
  • register_check.php:用于用户和管理员注册检测
    • 注册加密 传入数据库

首页模块


  • 主要用到Bootstrap自定义样式
    • 顶部导航栏吸顶效果
    • 栅格系统响应式布局
    • 移动端分辨率导航栏下拉

底部模块与侧边栏


  • 主要是实时查询数据库操作
    • mysql_fetch_assoc针对查询不到时的情况处理
    • 最新最热文章限制取出记录个数并按照关键字排序

评论模块

评论列表

  1. 构建查找匹配到对应id文章的评论表SQL语句
  2. mysql_fetch_row($result)取出记录个数
  3. 判断mysql_fetch_row($result)
    • 若不为空 mysql_fetch_assoc($result)遍历迭代出评论表
    • 若为空 不输出

发布评论

  1. 判断内容是否为空
    • 若不为空则构建SQL语句插入评论表中
    • 若为空则提示用户输入
  2. 与发布新闻模块一致用到了kindeditor富文本插件

新闻模块

发布新闻


因为数据库搭建的外键都是用到了各个表的ID，而实际上我们显示一半都是显示username，所以在对新闻表操作之前一般要构建SQL语句查找到对应的username或者id

  1. 根据表单传入内容，在顶部判断传入参数是否符合要求达到是否传入数据库的判断
  2. 与评论模块一致用到了kindeditor富文本插件

分页模块

  1. 构建查找出数据库所有符合SQL语句
  2. 设置每页显示的条数，由取出的记录算出分页的个数
  3. 每次从地址栏获取到 page(当前页面的页码) 和 startrow(数据库中根据当前页面应该显示的起始条数) 参数
  4. 由 page 和 startrow 参数在上下页以及切换到对应页面的位置设置a跳转

个人感悟

1.这是第一次接触PHP这类后台语言，对数据库操作语句返回值类型不太熟悉，踩了许多坑
2.在数据库一开始设计时一些参数数据类型设计不合理，导致后续查找语句中也踩了很多坑
3.用gulp等流程构建工具来开发提高了许多效率
4.对于模块的分割，一开始做了功虽然多，但是后期维护效率高
5.没有使用MVC等思想构建项目，后台框架不熟悉，在路由控制与模板渲染上手续繁杂，下次尝试
