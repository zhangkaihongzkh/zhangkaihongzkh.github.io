云主机搭载LAMP部署PHP项目

发表于 2018-02-28   |   分类于 躺过的坑   |   | 阅读量 次

摘要:
腾讯云主机搭载LAMP环境并部署PHP项目

最近尝试将自己项目部署上腾讯云服务器，现在记录部署流程以及犯过的错误。

犯过的错误

对ssh认证理解不够透彻

现在网上流行的基本都是ssh认证，包括工具winSCP、putty等工具。现在简述SSH认证过程。
1.第一次登录服务器时，将服务器的公钥传递过来，本地计算机进行存储，下次传递数据时将使用这个公钥对数据进行加密。
2.服务器接收到利用自己公钥加密的数据后，使用自己的私钥才能解开

数据库连接设置

由于是MySQL数据库配置的绑定地址是127.0.0.1(本机地址)，而且MySQL默认是关闭远程访问权限的。当我将项目部署到服务器上时，连接的是公网地址，所以无法连接到数据库。这时候需要
1.修改mysql数据库的user表，将host字段的值改为%，这样在任何客户端机器上能以root用户登录到mysql服务器
2.修改mysql配置文件vi /etc/mysql/mysql.conf.d/mysqld.cnf 将bind-address = 127.0.0.1注销​.
这样就是实现远程登录mysql，同时公网也能连接上数据了

没有安装php-mysql的扩展

导致处理逻辑时识别不了mysql语句，解决办法是安装相对应的包。

php版本

云主机一开始我使用的是ubuntu16.06，里面内置了PHP7.0，而PHP7.0放弃了很多PHP5.X不安全的语句，导致了语法错误。而下载相关的包时也会发现找不到。由于自己php并不打算深入学习，之前用的是php5.x的语法，这次主要是为了尝试部署流程，就把系统回退到14.04，安装相应的包。

工具

winSCP 用于本地文件与服务器文件交互，由于有图形化界面，对我这种新手十分友好。只需将文件拖入上传即可，使用的是ssh认证。

putty 用于远程登录服务器，操作服务器等，也可以使用腾讯云自带的终端系统。

navicat 用于远程登录数据库，也可以进入终端敲指令实现数据库的创建

修改mysql表与my.cnf配置文件

按照以上操作，服务器端的LAMP环境基本搭建完成。这时候就要去修改数据库mysql中的user表，同时设置php.ini文件绑定的默认地址

修改mysql表

1.登录mysq数据库

可以看到在user表中已创建的root用户。host字段表示登录的主机，其值可以用IP，也可用主机名，

2、实现远程连接(授权法)
将host字段的值改为%就表示在任何客户端机器上能以root用户登录到mysql服务器，建议在开发时设为%。

mysql> use mysql;
Database changed
mysql> grant all privileges on . to root@’%’ identified by “root”;
Query OK, 0 rows affected (0.00 sec)

mysql> select host,user,password from user;
+————–+——+——————————————-+
| host | user | password |
+————–+——+——————————————-+
| localhost | root | A731AEBFB621E354CD41BAF207D884A609E81F5E |
| 192.168.1.1 | root | A731AEBFB621E354CD41BAF207D884A609E81F5E |
| % | root | *A731AEBFB621E354CD41BAF207D884A609E81F5E |
+————–+——+——————————————-+

来查看mysql默认的端口3306是否开启，允许哪个ip使用，如果你发现，前面有127.0.0.1，就说明，3306端口只能本机ip使用

2.打开mysql配置文件vi /etc/mysql/my.cnf
需要注意的是，这个文件需要修改读写权限，修改完要改回其他用户不可写。否则会报错（又躺了一个坑）

‘/etc/my.cnf’ is ignored ，大概意思是权限全局可写，任何一个用户都可以写。mysql担心这种文件被其他用户恶意修改，所以忽略掉这个配置文件。导致无法进入，这也是mysql的安全机制之一。所以我们必须得改一些权限。设置其他用户不可写。

不同版本php的文件名不同，有的是在
/etc/mysql/mysql.conf.d/mysqld.cnf位置
