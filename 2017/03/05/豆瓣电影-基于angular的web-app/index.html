豆瓣api接口jsonp的适配

由于web不支持 对不同源的网址获取数据，而通过srcipt标签动态生成引入文件中，从而实现跨域请求数据的方式

豆瓣提供的api接口支持jsonp的调用，而angular提供的jsonp函数。angular为了避免造成全局变量污染，所以在所有方法前都带了angular对象

而豆瓣提供的api不支持带有angular对象的回调方式，所以要封装一个httpservice服务，提供jsonp方法供调用

jsonp具体实现

思路主要为下

  • 随机生成函数名称(确保每一次jsonp请求的回调函数都不一样)
  • 判断传入的data是否带有参数
    • 如果带有参数，则进行字符串的拼接到url中
    • 如果没有参数，则默认传入两个参数，为了适配，将data赋值给callback供回调
  • 拼接完成后，动态生成script节点
  • 在全局window中注册事件

  • 由于是基于angularu的单页应用，每次页面跳转进行的锚跳转，而整个外容器不变，这时候就有一个很恶心的问题了，每一次请求都会生成一个script节点

  • 所以我将注册事件放在回掉函数中，由于回调机制，默认会先注册事件，然后执行下一步

  • 将script节点插入页面中
    • 然后再来执行回调中注册的事件

这样 一个自制的带有jsonp函数服务就完成了，缺点是这样的机制和jq的$.ajax一样，是在全局中注册事件的，所以无可避免的造成了全局污染

jsonp和ajax的一些联系

鉴于之前阿里的面试曾经提及这个问题，之前我一直以为ajax和jsonp只是跨域不跨域的区别，后面发现我错了，在这里记录一下之间的ajax和jsonp联系与区别

先来谈谈为何会产生jsonp的技术吧

jsonp的产生

  • ajax不支持跨域请求
  • 而web端调用js文件不受跨域的影响
  • 所以形成了一个非正式的协议 – jsonp

ajax和jsonp的核心

  • ajax的核心是通过xmlHttpRequest对象来获取非本页内容数据
  • 而jsonp的核心是通过动态生成script标签引入到文本中调用服务器端脚本
  • 所以ajax和jsonp的核心不在于是否支持跨域 ajax通过服务器代理可以实现跨域请求，而jsonp自身也支持同源获取数据

电影详情

电影详情的接口格式为

通过传入不同的id来实现获取页面的详情，由于每个页面的id不同，所以在列表页面进入详情页面的时候，要将电影的id传到接口上，所以我通过指定详情页面路径的跳转，同时匹配id来实现跳转到详情页面

到这里就会有个问题了，通过接口的对比，可以发现详情页面和列表页面的api的路径结构是一样的

而在列表接口的api，我通过匹配参数来实现获取不同的接口
而在列表的路由分配中 会默认把subject也作为参数传到move_list模块中

我的解决方法是修改依赖的次序，将detail模块的至为list模块前，由于detail模块先加载，会先匹配到movie_detail模块的路由和控制器，进行渲染。而list的参数由于和detail参数不匹配，会进入movie_list模块进行渲染。达到不影响的效果
